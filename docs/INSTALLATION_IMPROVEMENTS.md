# ACE系统安装脚本改进说明

## 概述

本改进版本严格遵循 `install.sh` 中定义的版本要求，提供了高可靠性的安装体验。

## 版本要求（严格执行）

根据原 `install.sh` 文件，以下版本要求被严格执行：

- **Node.js**: 22.12.0 LTS (必须精确版本)
- **pnpm**: 10.10.0 (必须精确版本)
- **Python**: 3.13+ (最低要求)
- **npm**: 11.4.2+ (最低要求)

## 主要改进特性

### 1. 严格版本控制
- ✅ 精确版本验证和安装

- ✅ 版本不匹配时自动重新安装

### 4. 智能依赖管理
- ✅ 自动sudo权限管理
- ✅ 系统依赖自动安装
- ✅ Python虚拟环境自动创建

### 5. 详细的安装日志

## 使用方法

### 基本安装
```bash
# 使脚本可执行
chmod +x install_reliable.sh

# 运行安装
./install_reliable.sh
```

### 使用sudo权限安装
```bash
# 脚本会自动请求sudo权限
./install_reliable.sh
```

## 安装流程

1. **系统检测**: 自动检测操作系统和版本
2. **权限获取**: 安全获取sudo权限（需要用户输入密码）
3. **依赖安装**: 安装系统级依赖包
4. **Node.js安装**: 精确安装Node.js 22.12.0
5. **npm升级**: 确保npm版本≥11.4.2
6. **pnpm安装**: 精确安装pnpm 10.10.0
7. **Python检查**: 确保Python版本≥3.13
8. **项目设置**: 创建项目目录和虚拟环境
9. **依赖安装**: 安装前端和后端依赖
10. **环境配置**: 设置环境变量和权限

## 故障排除

### 常见问题和解决方案

#### 1. sudo权限问题
```bash
# 如果sudo权限获取失败
sudo -v  # 手动获取权限
./install_reliable.sh
```

#### 2. 网络连接问题
- 脚本会自动重试3次
- 检查网络连接状态
- 考虑使用代理或镜像源

#### 3. 版本冲突
- 脚本会自动卸载冲突版本
- 使用nvm管理Node.js版本
- 使用虚拟环境隔离Python

#### 4. 权限错误
- npm全局安装权限自动修复
- Python虚拟环境自动创建
- 文件权限自动设置

### 日志文件

安装日志保存在: `install_YYYYMMDD_HHMMSS.log`

查看最新日志:
```bash
ls -la install_*.log | tail -1
```

## 版本验证

安装完成后，脚本会自动验证所有版本:

```bash
# 验证Node.js版本
node --version  # 应该显示 v22.12.0

# 验证pnpm版本
pnpm --version  # 应该显示 10.10.0

# 验证Python版本
python3 --version  # 应该显示 3.13.x

# 验证npm版本
npm --version  # 应该显示 ≥11.4.2
```

## 安全特性

1. **密码安全**: sudo密码不在脚本中硬编码
2. **权限最小化**: 仅在需要时请求sudo权限
3. **错误隔离**: 每个安装步骤独立验证
4. **回滚机制**: 安装失败时提供清理建议

## 性能优化

1. **并行安装**: 依赖项并行下载和安装
2. **缓存利用**: 重复使用已下载的包
3. **智能重试**: 避免不必要的重复操作
4. **增量更新**: 仅更新必要的组件

## 维护和支持

### 更新版本要求
如果需要更新版本要求，请修改脚本顶部的版本常量:

```bash
NODEJS_REQUIRED_VERSION="22.12.0"
PNPM_REQUIRED_VERSION="10.10.0"
PYTHON_MIN_VERSION="3.13"
NPM_MIN_VERSION="11.4.2"
```

### 添加新平台支持
在 `install_system_deps()` 函数中添加新的操作系统支持。

### 调试模式
设置环境变量启用详细调试:
```bash
export DEBUG=1
./install_reliable.sh
```
