# MC/OC证据框架提取指南

## 英国GTV数字技术路径（Digital Technology）证据体系

### Mandatory Criteria（MC - 必要标准）

申请人必须证明自己是数字技术领域的**公认领导者**或**新兴领导者**。

| 编号 | 标准 | 证据方向 |
|------|------|----------|
| MC1 | 申请人在数字技术领域的专业成就和认可 | 个人奖项、媒体报道、行业认可 |
| MC2 | 申请人在产品/平台领域的领导角色 | 公司组织架构、产品宣传材料、领导层证明 |
| MC3 | 商业成功的证据 | 市场报道、收入增长、用户规模 |
| MC4 | 在行业中的影响力 | 数据统计、政府认可、市场份额 |

### Optional Criteria（OC - 可选标准）

申请人需至少满足2项可选标准。

| 编号 | 标准 | 证据方向 |
|------|------|----------|
| OC1 | 创新贡献 | 创新产品、技术突破、收入增长 |
| OC2 | 学术贡献 | 论文、专利、学术认可 |
| OC3 | 技术领导力 | 技术决策、团队建设、项目管理 |
| OC4 | 行业外贡献 | 社会影响、跨界合作 |

## 证据文件分析方法

### 当用户提供按MC/OC分类的证据目录时

使用脚本自动提取：

```bash
python3 scripts/extract_evidence.py "[证据目录]/" --json -o evidence_framework.json
```

脚本会根据文件名中的编号（如 MC2、OC1.1）自动归类，提取关键事实和申请人相关表述。

### 当用户仅提供综合材料时

手动从材料中按以下维度整理：

1. **公司/平台信息**：公司名称、规模、市场地位、技术平台功能
2. **政府/社会认可**：政府项目、服务规模、覆盖范围
3. **技术创新**：知识产权、技术突破、创新产品
4. **商业成就**：收入增长、市场份额、客户数据
5. **媒体报道**：主流媒体对申请人或公司的报道
6. **行业影响**：行业标准制定、演讲、论文、开源贡献

## 证据使用原则

1. **事实第一**：每条证据必须有原始材料支撑，标注出处文件
2. **不得杜撰**：如果材料中没有记载，不得编造数据或事实
3. **精准引用**：数据引用必须与原始材料完全一致
4. **上下文完整**：引用事实时保留必要的上下文，避免断章取义
5. **关联自然**：推荐人对证据的评价应与其专业领域自然关联
