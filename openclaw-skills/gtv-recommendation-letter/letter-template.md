# GTV推荐信写作模板与指南

## 推荐信结构模板

核心原则：**像一封自然流动的推荐信，而非分章节的报告**。

```
推荐信

[推荐人自我介绍段落：我是XXX，现任XXX。介绍专业背景、行业地位和相关成就。]

[推荐意愿段落：基于我的专业背景...非常荣幸地向贵委员会推荐[申请人]先生/女士。]

[认识经过段落：我与[申请人]先生/女士于XX年相识...描述合作经历。]

[专业能力评价段落群：连续多段自然段落，不加编号。
  - 技术创新与产品能力
  - 市场领导力与商业成果
  - 社会影响力与规模]

[行业认可段落：国家级项目、媒体报道、政府合作等。]

[英国发展潜力段落：市场需求、技术可迁移性、商业前景。]

[总结推荐段落：坚信[申请人]先生/女士符合标准，毫无保留地推荐。]

[推荐人中文姓名]
[中文职务1]
[中文职务2]
[中文单位名称]
电子邮箱：[邮箱]
[中文地址]
日期：[中文日期，如2026年X月X日]
```

### 格式禁忌清单

| 禁止项 | 说明 |
|--------|------|
| ❌ 独立头部块（日期/致/关于） | 直接以正文开始 |
| ❌ 编号章节（一、二、三） | 使用自然段落流 |
| ❌ 子编号（2.1, 2.2, 2.3） | 同上 |
| ❌ Markdown标记（`**粗体**`） | 使用Word原生加粗 |
| ❌ 出处列表 | 不放入正式信件，留在构思思路文件 |
| ❌ "此致/敬礼" | 使用中文签名落款块 |
| ❌ 双重签名块 | 签名块只出现一次 |

## 篇幅要求

**硬性限制：≤3页A4，正文2400~3000中文字符（仿宋12pt，1.5倍行距，每页约900字）**

- 低于2400字内容不够充实，必须补充
- 超出3000字必须精简

### 各段字数预算

| 段落 | 字数 | 要点 |
|------|------|------|
| 推荐人自我介绍 | 250~350 | 3-5项最权威的头衔/成就，适当展开核心履历 |
| 推荐意愿 | 50~80 | 自然过渡 |
| 认识经过 | 250~350 | 时间+背景+合作性质，适当展开合作故事细节 |
| 核心能力与成就 | 1000~1400 | 信的重心，2-3段，充分使用数据和具体事实论证 |
| 行业认可 | 250~350 | 合并为1段，选最有分量的2-3项，辅以具体细节 |
| 英国潜力 | 250~350 | 1段：需求+可迁移性+独特优势 |
| 总结推荐 | 150~200 | 重申判断，回顾关键数据，简洁有力 |

### 精炼写作原则

1. **数据优先于描述**：用具体数字说服，而非抽象形容
2. **一个论点一段话**：不为同一观点反复用多段论证
3. **删除空洞修饰**：去掉"令人瞩目的""极为突出的""尤为令人敬佩"等
4. **合并同类**：相关认可一句带过，不分段展开
5. **推荐人介绍精简**：只保留与推荐资格直接相关的信息
6. **不重复引用同一证据**：同一个数据点只出现一次

## 写作风格指南

### 语气与口吻
- 正式但不刻板，专业但有温度
- 以推荐人第一人称视角撰写
- 体现推荐人的行业权威性
- 对申请人的评价具体而非空泛

### 论证逻辑
- 先建立推荐人的可信度（为什么推荐人有资格评价）
- 再建立与申请人的关系（为什么推荐人了解申请人）
- 然后展开具体评价（基于事实和证据）
- 最后给出结论（推荐人的明确判断）

### 证据运用策略

根据推荐人的专业领域，对证据项进行差异化处理：

**如果推荐人来自科技/互联网行业**：
- 重点展开：技术创新、产品架构、AI应用
- 适度提及：市场成绩、服务规模
- 视角：从技术专业性角度评价

**如果推荐人来自实体行业/社会服务**：
- 重点展开：服务规模、社会影响、政府认可
- 适度提及：技术创新的具体功能和效果
- 视角：从行业需求和社会价值角度评价

**如果推荐人来自政府/政策研究领域**：
- 重点展开：政策影响、政府项目、监管创新
- 适度提及：技术平台的能力
- 视角：从政策创新和公共服务角度评价

**如果推荐人来自商业/投资领域**：
- 重点展开：商业成功、市场份额、增长数据
- 适度提及：技术创新和社会影响
- 视角：从商业价值和市场潜力角度评价

**如果推荐人来自学术/科研领域**：
- 重点展开：技术原创性、方法论创新、学术价值
- 适度提及：应用效果和商业转化
- 视角：从科研方法论和技术范式角度评价

### 弱项弥补策略（通用模板）

根据律师建议中识别的弱项，在推荐信中采用对应策略：

| 常见弱项 | 弥补策略 |
|----------|----------|
| 角色是否足够"技术性" | 推荐人作为知情人证明申请人深度参与技术决策和产品设计 |
| 外部媒体报道有限 | 推荐人从行业内部视角提供"业内人士"的认可 |
| 缺乏会议演讲/论文 | 强调实践成果比学术发表更能证明能力 |
| 缺乏国际知名度 | 推荐人阐述英国发展计划的可行性和市场需求 |
| 无个人专利 | 说明知识产权归属公司但由申请人主导开发 |
| 工作年限不足 | 强调短时间内取得的超常成就和影响力 |
| 频繁跳槽 | 从职业发展角度解释每次转变带来的能力提升 |

## 最终检查清单

完成推荐信后，逐项检查：

- [ ] 推荐人介绍信息准确，与网络搜索结果一致
- [ ] 与申请人的关系描述与推荐人信息文件一致
- [ ] 所有引用的事实均可在原始材料中找到
- [ ] 没有使用任何范本中的具体人物内容
- [ ] 使用中文撰写
- [ ] 论证逻辑清晰：推荐人资格→认识经过→具体评价→结论
- [ ] 针对律师提出的弱项有策略性回应
- [ ] 推荐信正文2400~3000字，≤3页A4
- [ ] 已保存为 docx 格式
- [ ] 构思思路文件已保存为 md 格式
- [ ] 所有文件位于 `推荐材料/[推荐人姓名]/` 目录下
